Skill 插件化方案

A. Skill 入 rp-core manifest

在现有 node.plugin.json 中增加 "skills" 字段，server 从所有 plugin 聚合 skill 列表

Pros

  • 最小改动，复用现有 plugin 加载流程
  • skill 和节点天然绑定同一插件
  • 插件启用/禁用时 skill 同步生效

Cons

  • skill 和节点耦合在一个 manifest 里
  • 纯 skill 插件也需要声明空的 nodes 数组

B. 独立 skill 插件

新建 skill.plugin.json manifest 类型，skill 和节点插件彻底分离

Pros

  • skill 和节点独立管理、独立启用/禁用
  • 第三方可以只发布 skill 包

Cons

  • 新增 manifest 类型、加载逻辑、API
  • 前期设计工作量更大
