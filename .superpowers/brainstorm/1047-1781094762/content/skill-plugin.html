方案 B：skill 与节点插件分离，各自独立管理

与 node.plugin.json 的对比

node.plugin.json
nodes[] ← 节点定义
executor{} ← 执行入口
permissions[] ← 权限声明
~~ skills[] ~~ ← 不放在这
skill.plugin.json
skills[] ← 技能定义
~~ executor ~~ ← 无执行逻辑
~~ permissions ~~ ← skill 不需要权限
compatibility{} ← 版本兼容

加载与聚合流程

1. 扫描 — serve.mjs 扫描 plugins/*/skill.plugin.json
↓
2. 校验 — validateSkillPluginManifest() 校验 schema
↓
3. 合并 — 跳过 disabled，跳过无效，合并所有 skills[]
↓
4. 暴露 — GET /api/skills | GET /api/plugins (含 skills)
↓
5. 消费 — Agent 节点 config 面板通过 API 获取可选 skill 列表

API 设计

GET /api/skills — 返回所有启用的 skill

{
  "skills": [
    { "id": "rp_persona", "label": {...}, "content": {...},
      "category": "roleplay", "pluginId": "awp.rp-skills" },
    ...
  ],
  "categories": ["roleplay", "safety", "writing", "memory"]
}

GET /api/plugins — 扩展返回 skill 插件信息

{
  "id": "awp.rp-skills",
  "kind": "skill-plugin",        // 新增 kind 字段区分类型
  "skillCount": 7,
  ...
}
