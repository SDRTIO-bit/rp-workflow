## How to use this guide

Three batches, strictly sequential. Do NOT start A-2 until A-1's `npm --workspace
@awp/card-import test` is green and `npm run typecheck` is clean. Do NOT start
A-3 until A-2's E2E (`routes/cards.test.ts` L11-L15) is green.

Every entry below has: **path** · **role** · **owner package/app** ·
**new/modified** · **batch**. Files are grouped by batch then by concern.

## Explicitly NOT touched (guardrail)

These files must NOT change in P-15.3A — changing them violates the
architectural constraints:

- `packages/rp-runtime/**` — RP runtime unchanged
- `packages/agent-runtime/src/profileRegistry.ts` — no profile/prompt change
- `packages/workflow-worldbook/**` — store interface reused as-is
- `data/workflows/rp-unified-stateful-production-v1.json` — workflow JSON
  unchanged
- `plugins/rp-core/node.plugin.json` — `rpCharacterCard` node untouched;
  cards do NOT route through it (see report §C.3)
- Writer / Critic / Curator prompt templates — unchanged
- Any real character card file — never enters Git (`.gitignore` covers
  `data/`)

## Dependency direction (must hold)

```
@awp/workflow-core        (foundation)
   ▲
@awp/workflow-worldbook   (DynamicWorldbookStore, DynamicWorldbookEntryV1, normalizeEntry)
   ▲
@awp/card-import          (NEW — pure logic, no HTTP)
   ▲
@awp/server               (routes/cards.ts, services/cardImportService.ts)
   ▲
@awp/web                  (features/cards/** — consumes API only)
```

`@awp/card-import` does NOT depend on `rp-runtime`, `agent-runtime`, or any
app. It is RP-agnostic and reusable.
