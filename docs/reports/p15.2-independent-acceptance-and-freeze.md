## 3. Acceptance Gap Closed

### 3.1 What the design flagged (Rev 2 §10.6)

The Rev 2 design's deterministic-replay fixtures were the correctness
check for V1. Pre-`9394493`, the existing `rpUnifiedProductionE2E` and
`rpSideEffectSafetyE2E` fixtures used <64-char writer texts, so
`textNoveltyCheck` never reached `exact_duplicate` inside an integrated
replay. The novelty → merge → route → writer2 → merge2 → selector →
side-effect chain was therefore **not** exercised end-to-end against the
real `rp-unified-stateful-production-v1.json` workflow JSON.

This is the §8/§10 acceptance gap the design left for the validation
phase.

### 3.2 What `9394493` adds

`packages/agent-runtime/src/rpNoveltyReplayE2E.test.ts` (926 lines) uses
a 156-char Chinese narrative (the Turn-13/14 case from the design) so
the full chain actually fires against the real workflow JSON. The suite
covers:

- **Replay 0** — first turn, empty session → `no_reference` → accept
- **Replay A** — duplicate writer1 → novelty revise → writer2 fresh → accept
- **Replay B** — duplicate writer1 + writer2 → exhausted return-latest
- **Replay C** — duplicate writer1 + writer2 → exhausted fail
- **Replay D** — novelty pass, critic hard reject → revise/exhausted
  (novelty does not override critic; critic remains orthogonal)
- **Prompt capture** — Writer 1/2/Critic 1/2 content assertions
  (the `## Data` channel renders only the four `decision` fields; no
  gate scores, no sources, no `awp.rp-decision-diagnostics.v1` payload;
  Writer 2 sees only the workflow-JSON-configured
  `noveltyRevisionInstruction` exactly once)
- **Checkpoint/resume** — `merge1` checkpointed; on resume, `writer1`
  is not re-run, `writer2` continues, session and memory are not
  duplicated, and skipped nodes (e.g. on the Replay B exhausted path)
  stay skipped

Replay scenarios match the routing table in the design §7.3
(`exact_duplicate` on attempt 1 → revise with novelty's revision
instruction; `exact_duplicate` on attempt 2 → exhausted →
`rpSideEffectDecision` decides side effects per `behavior.onExhausted`).

## 8. Outstanding Warnings

1. **W-1 (cross-turn narrative repeat, inherited from P-15.1, mitigated by P-15.2)** — turn 14 of the P-15.1 20-turn run was byte-identical to turn 13 (156 normalized chars). P-15.2 V1 closes this by adding the deterministic `textNoveltyCheck` node. A future real-Provider re-run is the empirical proof; the deterministic-replay fixture in `rpNoveltyReplayE2E.test.ts` is the unit-level proof. Per project policy (`rp-workflow-comfyui-constraint` memory), any further mitigation is workflow / node / parameter work, not a code change to the LLM adapter or profile. Confirm with the user before any code change.
2. **W-2 (Writer 2 `data` port carries `schemaId` in `## Data`)** — the rendered Writer 2 prompt now contains the `awp.rp-decision.v1` `schemaId` field as part of the `## Data` JSON block. This is informational, not a contract violation, and is the same pattern P-15.1 used for the `gate1.result` JSON. A future preset that wants to hide the `schemaId` would do so via prompt-template work, not a code change. Non-blocking.
3. **N-1 (session `tokenUsage` estimate, inherited from P-15.1)** — session-store `tokenUsage` is a small-integer estimate, not Provider observability. Internal to the session store, does not affect billing or observability contracts. Non-blocking.
4. **N-2 (no per-role input / output token breakdown, inherited from P-15.1)** — `run-summary.json` records aggregate per-turn tokens, not per-call per-role. A future phase may add a per-role telemetry node (workflow-layer, not runtime-layer). Non-blocking.

## 9. Out of Scope (explicit)

The following were deliberately not built in P-15.2 V1. They are not
claims of this report.

- **Near-duplicate similarity as a forced revise trigger.** V1.1
  follow-up, contingent on a labeled corpus and threshold calibration.
  The `reason` enum in `awp.text-novelty-report.v1` reserves no
  near-duplicate value; `exactDuplicate` is the only verdict field.
- **Embedding / vector / LLM-judge similarity.** Explicitly excluded
  by the Rev 2 design.
- **Window > 1 for reference text.** V1 uses the most recent committed
  turn only.
- **Multi-locale normalization (Japanese kana folding, Arabic
  diacritics, etc.).** V1 is language-neutral by design.
- **A new `rpSideEffectDecision` policy for "novelty-exhausted ⇒ no
  commit".** The existing `behavior.onExhausted` is the only exhausted
  side-effect policy. If the user wants a stricter rule, that is a
  separate ticket.
- **A third Writer / Critic pass.** Excluded.
- **Bilingual revision instructions.** V1 official RP is Chinese; a
  single Chinese string is the default.
- **Lowering `minNormalizedLength` below 64.** V1 default is 64.
  Workflow authors may lower it via the JSON; that is a workflow-author
  choice, not a code change.
- **Auto-curating novelty reports into long-term memory.** Curator
  sees only the final accepted draft, unchanged.
- **Real-Provider re-run of the 20-turn corpus.** Inherited from
  P-15.1; not redone for P-15.2 freeze.
