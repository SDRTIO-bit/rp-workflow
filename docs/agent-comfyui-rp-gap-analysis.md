# Agent Workflow Platform 差距分析与 RP 节点方案

日期：2026-06-10

## 对比 ComfyUI 的不足

当前项目已经有自研画布、节点、端口、连线和基础运行能力，但距离 ComfyUI 级别的成熟节点系统还有明显差距：

- 节点生态不足：ComfyUI 的优势是大量内置节点、社区节点、模型加载器、采样器、条件控制、预览和后处理链路。本项目目前节点数量少，更多是产品原型节点。
- 节点参数面板较弱：目前只按节点类型手写少量表单，没有统一 schema 渲染、参数校验、默认值说明、枚举、范围滑杆、条件字段和高级折叠区。
- 缺少节点搜索与分类体验：已有分类节点库，但还没有搜索、收藏、最近使用、拖拽创建、节点帮助文档和快捷插入。
- 缺少执行可视化：ComfyUI 对执行排队、节点状态、预览结果、错误定位更直观。本项目只有运行记录和节点状态，缺少每个节点的输入输出快照。
- 缺少中间结果预览：虽然已补 `assetPreview` 和检查器预览，但还不是每个节点都有真实输出预览、展开查看、复制结果和历史版本。
- 缺少图级操作：尚未支持框选、多选、对齐、分组、折叠、复制子图、模板化子图、局部运行和断点运行。
- 缺少插件生命周期：目前是 registry 驱动的“外来扩展节点”形态，但还没有真正的插件安装、版本、权限、依赖、禁用、迁移机制。
- 缺少队列与缓存策略 UI：Agent/LLM 调用尤其需要缓存命中、token、耗时、失败重试、成本估算和可复现输入快照。

## 面向 Agent 能力的不足

Agent 工作流和图像工作流的差别在于：Agent 节点需要状态、工具、记忆、审批和可解释轨迹。当前不足主要在这些方向：

- Agent 状态模型不完整：节点只有 `config` 和边输入，缺少长期状态、会话状态、角色状态、任务状态和跨轮上下文压缩策略。
- 工具调用不可见：现在插件/skill 更多是 config 字段，没有真实工具调用链、工具入参出参、权限确认和失败恢复。
- 多 Agent 协作协议较薄：已有并行 Agent，但缺少角色分工、仲裁、投票、反思、planner/executor/reviewer 等常见协作模式。
- 缺少人类审批节点：没有“暂停等待用户确认”“编辑中间结果后继续”“批准工具调用”等 Agent 产品常见节点。
- 缺少上下文预算管理：没有 token 预算、引用来源、上下文裁剪、记忆召回权重和 prompt 组成可视化。
- 缺少评估节点：没有自动检查事实、一致性、安全边界、格式合规、任务完成度的 eval/critic 节点体系。
- 缺少可恢复运行：失败后无法从某个节点恢复，也没有持久化 run session、trace id、重放和对比运行。
- 缺少真实外部连接器：OpenAI、浏览器、文件、数据库、Slack、GitHub 等连接器目前只是节点语义，没有成熟执行器。

## RP 角色扮演补充方案

本轮新增了 RP 专用节点和参考工作流，定位为“外来扩展节点”：

- `rpCharacterCard`：角色卡节点，输出 `character_profile`。
- `rpSceneState`：场景状态节点，输出 `scene_state`。
- `rpLoreRecall`：设定召回节点，输出 `search_result`。
- `rpDialogueDirector`：对话导演节点，整合角色、场景、玩家输入和记忆，输出 RP 回复草稿。
- `rpContinuityCheck`：连续性检查节点，检查人设、世界观、关系状态和玩家行动权。

新增参考工作流：

- `roleplayWorkflow`，位置：`apps/web/src/state/sampleWorkflows.ts`
- 前端顶部新增“加载 RP 示例”按钮。

参考链路：

```text
玩家输入
  -> RP 角色卡
  -> RP 设定召回
  -> RP 场景状态
  -> RP 对话导演
  -> 正文输出
  -> RP 连续性检查
```

## 下一步建议

优先级建议如下：

1. 给 RP 节点补真实 Agent executor，把角色卡、场景状态、设定召回和连续性检查接入真实 LLM。
2. 给每个节点增加输入输出快照面板，先做 Agent/RP 节点，再扩展到所有节点。
3. 给节点定义补统一参数 schema，让检查器不再靠节点类型手写表单。
4. 增加节点搜索、拖拽创建、多选删除和框选操作，补齐与 ComfyUI 的基础体验差距。
5. 给 Agent 节点补成本、token、cache hash、trace id 和可恢复运行。
